# 视觉小说翻译器 - 窗口选择界面需求文档

## 1. 概述
本文档描述视觉小说翻译器的第二个界面——窗口选择界面的功能与交互需求。该界面在用户完成翻译设置后显示，用于选择需要监听的游戏/应用程序窗口。

## 2. 界面布局与功能

### 2.1 窗口标题
- 标题文字：**选择目标窗口**

### 2.2 引导文案
- 文案内容：*“请从下方列表中选择要进行OCR识别的窗口”*
- 位置：界面顶部，下拉框上方

### 2.3 窗口选择区域
- **窗口下拉选择框**
  - 控件类型：下拉选择框（Dropdown）
  - 功能：显示当前可用的窗口列表
  - 列表项格式：`窗口标题 - 进程名`（例如：`Summer Pockets - summer.exe`）
  - 默认状态：无选中项，或显示“请选择窗口”的占位提示
  - 限制：只显示可见窗口、有标题的窗口、非最小化窗口（通过Win32 API过滤）, 可以考虑实现过滤掉当前自己的窗口

- **刷新按钮**
  - 控件类型：普通按钮
  - 按钮文字：🔄 刷新
  - 功能：重新枚举当前所有符合条件的窗口，更新下拉列表
  - 交互：点击后下拉列表内容立即刷新，之前的选择清空

### 2.4 底部操作按钮
- **上一步**
  - 功能：返回翻译设置界面，当前窗口选择的进度不保存

- **下一步**
  - 功能：记录选中的窗口句柄（HWND），进入下一界面
  - 状态：仅当下拉框有选中项时，按钮才可点击（否则置灰禁用）
  - 异常处理：如果选中窗口在点击下一步前已关闭，提示用户重新选择

## 3. 交互逻辑补充

### 3.1 窗口列表加载
- 进入界面时自动加载一次窗口列表
- 若扫描结果为0个窗口，下拉框显示“未找到可用窗口”，刷新按钮保留

### 3.2 窗口选中反馈
- 用户选中某个窗口后，可以在下拉框旁边显示该窗口的简要信息（如窗口尺寸），帮助用户确认选择正确

### 3.3 窗口变化处理
- 用户长时间停留在此界面时，不自动刷新窗口列表（避免用户选中后列表变化导致困惑）
- 由用户主动点击“刷新”按钮更新列表

## 5. 数据传递
- 向下传递：无（从翻译设置界面继承配置）
- 向上传递：选中的窗口句柄（HWND），用于后续的截屏区域定位
- 保存内容：进入下一界面时，需要保存选中的窗口句柄，后续截屏操作将基于该窗口进行
