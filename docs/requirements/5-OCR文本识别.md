# 视觉小说翻译器 - OCR识别后台功能需求文档

## 1. 概述
本文档描述视觉小说翻译器的后台OCR识别功能需求。该功能在用户完成所有配置（翻译设置、窗口选择、区域选择）后启动，在后台持续运行，无需用户直接交互。其核心任务是对目标窗口的指定区域进行周期性截图检测，当检测到内容发生变化时触发OCR识别，并将识别结果传递给后续翻译模块。

## 2. 功能模块与流程

### 2.1 定时截图模块
- **触发方式**：以固定时间间隔自动执行（例如每秒3-5次，可根据性能调整）
- **截图对象**：用户在前序步骤中选定的目标窗口
- **截图区域**：基于用户框选的相对位置，从窗口截图中裁剪出指定区域
  - 每次截图时，根据窗口当前的实际尺寸，将保存的相对比例坐标转换为绝对像素坐标
  - 例如：相对区域 {left:0.25, top:0.70, right:0.75, bottom:0.85}，当前窗口尺寸1280x720，则实际裁剪区域为(320, 504) 到 (960, 612)
- **截图格式**：内存中的图像数据（如RGB格式），无需保存为文件

### 2.2 图像相似度判断模块
- **功能**：比较当前截图与上一次处理过的截图是否相似，决定是否需要进行OCR
- **比较对象**：
  - 首次运行时，无上一次截图，直接进行OCR
  - 后续每次截图，与“上一次进行OCR的截图”进行比较（注意：不是与上一次截图比较，而是与最后一次触发OCR的那张图比较）
- **相似度算法**：使用pHash（感知哈希算法）或其他适合的图像相似度算法
- **判断逻辑**：
  - 计算当前截图与上一次OCR截图的哈希值，计算汉明距离
  - 设定一个阈值（例如汉明距离 < 10 视为相似），当差异超过阈值时判定为内容变化
  - 阈值应支持配置，以适应不同游戏的字幕变化速度
- **性能要求**：相似度计算必须高效，不能明显影响截图频率

### 2.3 OCR识别模块
- **触发条件**：当相似度判断模块判定图像内容发生变化时
- **识别内容**：当前截取的图像区域
- **识别语言**：用户在翻译设置界面选择的源语言（日语/英语）
- **输出结果**：识别出的文本字符串
- **异常处理**：
  - 识别失败（如图像模糊、文字过小）时，返回空结果或错误码
  - 连续失败达到一定次数后，可适当降低截图频率或告警（但后台模块不直接展示告警，仅记录日志）

### 2.4 数据传递
- **输出传递**：OCR识别出的文本（无论是否为空）传递给后续的翻译模块
- **状态记录**：记录本次OCR使用的截图，作为下一轮相似度比较的依据

## 3. 配置参数（可后续在设置界面提供）
- **截图间隔**：默认值建议1秒
- **相似度阈值**：默认值建议根据pHash特性设定（如汉明距离≤10），支持调节
- **OCR语言**：继承自翻译设置界面，不可在后台单独修改

## 4. 逻辑流程示意图

---------------------------------------------
| OCR识别后台逻辑流程                         |
---------------------------------------------
| 启动后台任务                                 |
|      ↓                                       |
| 从配置读取：窗口句柄、选区比例、源语言          |
|      ↓                                       |
| 初始化：上一次截图 = null                      |
|      ↓                                       |
| 进入循环(每隔N毫秒执行一次)                    |
|      ↓                                       |
| 获取目标窗口当前尺寸                           |
|      ↓                                       |
| 根据相对比例计算实际裁剪区域                     |
|      ↓                                       |
| 对指定区域进行截图 → 当前截图                    |
|      ↓                                       |
| 判断：上一次截图是否为null？                     |
|      ├─ 是 → 直接进入OCR流程                    |
|      └─ 否 → 计算当前截图与上一次截图的pHash相似度 |
|           ↓                                   |
|          判断：相似度是否超过阈值？                 |
|           ├─ 否 → 跳过，等待下一个周期            |
|           └─ 是 → 进入OCR流程                    |
|                ↓                               |
|               调用OCR接口识别当前截图文本           |
|                ↓                               |
|               将识别出的文本发送给翻译模块           |
|                ↓                               |
|               更新：上一次截图 = 当前截图           |
|                ↓                               |
|        等待下一个周期                             |
---------------------------------------------

## 5. 边界条件与异常处理

### 5.1 窗口状态变化
- **窗口最小化**：无法截图时，暂停循环，等待窗口恢复
- **窗口关闭**：停止后台任务，通知前端用户重新选择窗口
- **窗口尺寸变化**：每次截图时重新计算裁剪区域，无需额外处理

### 5.2 OCR失败处理
- OCR接口调用失败（如网络问题、API密钥失效）：
  - 记录错误日志
  - 本次不更新“上一次截图”，避免因一次失败导致后续一直无法触发
  - 连续失败达到一定次数后，可降低截图频率或暂停任务

### 5.3 性能考虑
- 截图和相似度计算应尽量轻量，避免占用过多CPU
- 可考虑多线程设计：截图线程与OCR处理线程分离，截图线程只负责截图和相似度判断，OCR操作在独立线程中执行，避免阻塞后续截图
