# 视觉小说翻译器 - 整体功能设计概述

## 1. 项目背景与目标
本项目旨在开发一款针对视觉小说游戏的实时翻译工具。通过屏幕识别（OCR）技术捕获游戏中的日文/英文文本，调用翻译API转换为中文，并以悬浮窗口形式实时展示给用户。整个流程自动化运行，最小化对游戏体验的干扰。

## 2. 整体功能流程

---------------------------------------------
| 视觉小说翻译器 - 完整工作流程                |
---------------------------------------------
| 启动软件 → 翻译设置界面                      |
|      ↓                                       |
| 窗口选择界面 → 选择目标游戏窗口                |
|      ↓                                       |
| 截图区域选择界面 → 框定文字识别区域            |
|      ↓                                       |
| 运行参数配置界面 → 设置识别/翻译参数           |
|      ↓                                       |
| 点击“开始翻译” → 进入自动运行模式              |
|      ↓                                       |
| 后台OCR模块持续监控指定区域                    |
|      ↓                                       |
| 内容变化时触发OCR识别 → 文本去重判断            |
|      ↓                                       |
| 调用翻译API → 结果显示在悬浮窗                 |
---------------------------------------------

## 3. 功能模块划分

### 3.1 配置引导模块（界面类）
包含四个顺序出现的配置界面，引导用户完成所有必要设置：

| 界面名称 | 核心功能 | 输出数据 |
|---------|---------|---------|
| 翻译设置界面 | 选择源语言、翻译引擎、配置API密钥 | 源语言、引擎类型、密钥 |
| 窗口选择界面 | 从系统窗口列表中选择目标游戏窗口 | 窗口句柄(HWND) |
| 截图区域选择界面 | 在截图上框选文字识别区域 | 选区相对比例坐标 |
| 运行参数配置界面 | 设置OCR频率、阈值、显示样式 | 各项运行参数 |

### 3.2 后台服务模块（非界面类）
配置完成后自动运行，用户无直接交互：

| 模块名称 | 核心功能 | 输入 | 输出 |
|---------|---------|-----|-----|
| 定时截图模块 | 按设定间隔截取目标窗口指定区域 | 窗口句柄、选区比例 | 区域截图 |
| 图像相似度判断模块 | 比较截图变化，决定是否触发OCR | 当前截图、历史截图 | 是否触发OCR |
| OCR识别模块 | 调用OCR引擎识别图像文字 | 区域截图、源语言 | 识别出的文本 |
| 文本去重模块 | 比较文本变化，决定是否触发翻译 | 当前文本、历史文本 | 是否触发翻译 |
| 翻译引擎调用模块 | 调用阿里云/Google翻译API | 文本、密钥、语言对 | 翻译结果 |
| 悬浮窗显示模块 | 实时展示翻译结果 | 原文+译文、样式参数 | 用户可见的翻译结果 |

### 3.3 数据流向关系

---------------------------------------------
| 模块间数据传递关系                          |
---------------------------------------------
| 配置界面 → 保存参数 → 后台模块读取            |
|                                             |
| 定时截图模块 → 区域截图                      |
|        ↓                                    |
| 图像相似度判断 → (变化时) → OCR识别模块       |
|                             ↓                |

## 4. 关键设计要点

### 4.1 相对位置定位
- 截图区域使用相对于窗口的**比例坐标**（0-1之间的浮点数）而非绝对像素
- 无论窗口如何调整大小，都能自动计算正确的裁剪区域

### 4.2 多级去重机制
- **图像级去重**：pHash比较截图相似度，避免相同画面反复OCR
- **文本级去重**：比较OCR结果文本相似度，避免相同文本反复翻译
- 目的：减少不必要的API调用，提升性能，避免界面闪烁

### 4.3 状态化管理
- 配置界面与运行界面分离
- 开始翻译后锁定配置参数，暂停后恢复可编辑
- 悬浮窗口独立运行，支持拖动和缩放

### 4.4 异步处理
- OCR识别、翻译API调用均异步执行
- 避免耗时操作阻塞截图循环

## 5. 用户交互路径

```
启动软件
→ 翻译设置（语言/引擎/密钥）
→ 选择游戏窗口
→ 框选文字区域（类裁剪工具）
→ 设置运行参数（阈值/字体/透明度）
→ 点击“开始翻译”
→ 悬浮窗显示翻译结果
→ 可随时“暂停翻译”调整参数
→ 可拖动/缩放悬浮窗
```
