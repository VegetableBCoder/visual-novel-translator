# 视觉小说翻译器 - 截图区域选择界面需求文档

## 1. 概述
本文档描述视觉小说翻译器的第三个界面——截图区域选择界面的功能与交互需求。该界面在用户完成窗口选择后显示，用于在目标窗口的截图上框定需要进行OCR识别的文字区域。程序将记录该区域在窗口中的相对位置，以适应窗口尺寸变化。

## 2. 界面布局与功能

### 2.1 窗口标题
- 标题文字：**选择识别区域**

### 2.2 引导文案
- 文案内容：*“请在下方截图上框选需要OCR识别的文字区域”*
- 位置：界面顶部，图片上方

### 2.3 截图显示区域
- **功能**：显示目标窗口的当前截图，作为选区操作的画布
- **图片处理**：
  - 截图按比例缩放到适应窗口显示区域，保持原始宽高比
  - 图片周围如有空白区域，用中性色填充
  - 显示图片原始尺寸信息（例如：`原始截图: 1280x720`）在图片角落或状态栏

- **遮罩效果**：
  - 未被选中的区域覆盖半透明黑色遮罩（例如60%透明度）
  - 被选中的矩形区域内保持原始亮度，形成“裁切预览”效果
  - 遮罩和选区实时更新，跟随鼠标操作

### 2.4 选区操作交互
- **选区初始状态**：
  - 进入界面时，默认在图片中央显示一个预设大小的选区（例如图片宽高的60%）
  - 选区有可见的矩形边框（例如2像素宽的亮蓝色实线）

- **鼠标拖拽创建选区**：
  - 用户可以在图片任意位置按下鼠标左键，拖拽出新的矩形选区
  - 拖拽过程中实时显示矩形框，松开鼠标完成创建

- **选区调整**：
  - **移动**：鼠标移动到选区内部时，光标变为四向箭头，按下左键拖拽可移动整个选区
  - **拉伸**：选区边框上显示8个控制点（四个角+四条边中点）
    - 鼠标移动到控制点上时，光标变为双向箭头，指示拉伸方向
    - 按下左键拖拽控制点，可沿对应方向拉伸选区
    - 拉伸过程中保持对边/对角的位置联动

- **选区视觉反馈**：
  - 鼠标悬停在选区边框或控制点上时，边框高亮显示（例如变为更亮的颜色或加粗）
  - 选区操作过程中（移动/拉伸）实时更新遮罩区域和选区位置
  - 选区边缘保持清晰可见，不与背景混淆

- **选区边界限制**：
  - 选区不能超出图片边界
  - 选区最小尺寸限制（例如最小30x30像素），防止过小无法操作

### 2.5 操作按钮区域

- **重新截屏按钮**
  - 控件类型：普通按钮
  - 按钮文字：🔄 重新截屏
  - 功能：重新对已选中的目标窗口进行截图，更新界面中的显示图片
  - 适用场景：游戏内容变化、对话框位置移动、窗口调整大小后需要重新截图
  - 交互：点击后立即执行截图，图片刷新，选区重置为默认大小和位置

- **底部操作按钮**
  - **上一步**：返回窗口选择界面，当前选区设置不保存
  - **下一步**：保存当前选区的相对位置信息，进入下一界面（或完成引导）
  - **下一步按钮状态**：只要有有效的选区（不论是否默认），按钮即可点击

### 2.6 信息提示区域
- 显示当前选区的信息，帮助用户确认：
  - 选区尺寸：`宽度: 640px, 高度: 180px`
  - 相对位置：`左: 25.0%, 上: 70.0%, 右: 75.0%, 下: 85.0%`
  - 信息实时更新，随选区变化而变化

## 3. 交互逻辑补充

### 3.1 界面加载
- 进入界面时，自动对已选中的目标窗口进行一次截图并显示
- 截图失败时（如窗口已关闭），显示错误提示并禁用下一步，提供返回上一步或重新截屏的选项

### 3.2 选区操作优先级
- 鼠标操作需要明确区分：新建选区 vs 移动/拉伸已有选区
- 规则：点击空白区域（遮罩区）开始新建选区；点击选区内部或控制点进行移动/拉伸

### 3.3 键盘辅助（可选）
- ESC键：取消当前操作（移动/拉伸），选区回到操作前状态
- Delete键：删除当前选区，恢复全屏遮罩状态（便于重新框选）

### 3.4 数据记录
- 最终保存的不是像素值，而是**相对于图片原始尺寸的比例坐标**
- 格式示例：`{“left”: 0.25, “top”: 0.70, “right”: 0.75, “bottom”: 0.85}`
- 这些比例值将用于后续对任意尺寸的窗口截图进行区域裁剪
