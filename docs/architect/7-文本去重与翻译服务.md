# 文本翻译与结果展示模块 - 详细技术设计文档

## 1. 模块概述
文本翻译与结果展示模块是视觉小说翻译器的下游模块，负责接收OCR识别出的文本，进行文本去重、调用翻译API，并将翻译结果通过悬浮窗口实时展示给用户。该模块运行于OCR/翻译线程中，与OCR识别模块共享同一线程，以减少线程切换开销。

## 2. 文本去重模块设计

### 2.1 职责
- 维护上一次成功翻译的文本。
- 对当前OCR识别出的文本进行有效性过滤（空文本、纯标点等）。
- 计算当前文本与上一次文本的相似度，根据阈值判断是否为新文本，从而决定是否触发翻译。

### 2.2 类设计：Text
```python
class TextDeduplicator:
    def __init__(self, threshold=80):
        self.threshold = threshold  # 相似度阈值（0-100），默认80
        self.last_text = ""        # 上一次翻译的文本

    def should_translate(self, new_text: str) -> bool:
        1. 空文本过滤
           if not new_text or not new_text.strip():
               return False

        2. 纯标点符号过滤（可选，提高准确性）
           if self._is_punctuation_only(new_text):
               return False

        3. 首次翻译
           if not self.last_text:
               self.last_text = new_text
               return True

        4. 计算相似度
           similarity = self._calculate_similarity(self.last_text, new_text)

        5. 判断是否超过阈值（相似度低表示变化大，需要翻译）
           if similarity < self.threshold:
               self.last_text = new_text
               return True
           else:
               return False

    def _calculate_similarity(self, text1: str, text2: str) -> float:
        使用Levenshtein距离计算相似度
        返回0-100的数值，100表示完全相同
        pass

    def _is_punctuation_only(self, text: str) -> bool:
        检查是否只包含标点符号或特殊字符
        pass

    def reset(self):
        self.last_text = ""
```

### 2.3 相似度算法实现要点
- 使用 `python-Levenshtein` 库的 `ratio()` 方法，返回0-1之间的浮点数，乘以100得到百分比。
- 若不想引入外部库，可使用 `difflib.SequenceMatcher`，但性能稍差。
- 对于日语文本，可考虑先进行标准化（如全角半角转换、统一汉字变体），但需求中未要求，暂不实现。

### 2.4 标点符号过滤
- 定义标点符号集合：`！？。、，．：；」』」】）}]` 等。
- 去除所有标点后若剩余字符为空，则视为无效文本。

## 3. 翻译结果数据类设计

### 3.1 TranslationResult 数据类
```python
from dataclasses import dataclass

@dataclass
class TranslationResult:
    success: bool              # 翻译是否成功
    translated_text: str      # 翻译后的文本
    error: str = ""           # 错误信息（失败时）
    raw_data: dict = None     # 原始返回数据，用于调试
```

## 4. 翻译引擎模块设计

### 4.1 职责
- 根据用户配置的引擎类型（阿里云/Google）和API密钥，调用对应翻译API。
- 处理API调用的超时、异常，返回翻译结果或错误信息。
- 支持源语言（日语/英语）到简体中文的翻译。

### 4.2 类设计：Translator

```python
class Translator:
    def __init__(self, config_helper: ConfigHelper):
        self.config = config_helper
        self._engine_type = None
        self._init_engine()

    def _init_engine(self):
        从配置读取引擎类型和密钥
        self._engine_type = self.config.get("translation", {}).get("engine")

    def translate(self, text: str, source_lang: str, target_lang="zh", timeout=3) -> TranslationResult:
        返回格式：TranslationResult(success, translated_text, error)

        if not self._engine_type:
            return TranslationResult(success=False, translated_text="", error="翻译引擎未配置")

        try:
            if self._engine_type == "alibaba":
                return self._translate_alibaba(text, source_lang, target_lang, timeout)
            elif self._engine_type == "google":
                return self._translate_google(text, source_lang, target_lang, timeout)
            else:
                return TranslationResult(success=False, translated_text="", error="未知翻译引擎")
        except TimeoutError:
            return TranslationResult(success=False, translated_text="", error="翻译超时")
        except Exception as e:
            return TranslationResult(success=False, translated_text="", error=str(e))
```

### 4.3 阿里云翻译实现
- 使用 `requests` 库直接调用 HTTP API
- API 端点：`https://mt.cn-hangzhou.aliyuncs.com`
- 签名算法：按照阿里云 API 签名规范（HMAC-SHA1）
- 请求方法：POST
- 请求参数：
  - `Action`: `TranslateGeneral`
  - `Format`: `JSON`
  - `Version`: `2018-10-12`
  - `AccessKeyId`: 从配置读取
  - `SignatureMethod`: `HMAC-SHA1`
  - `SignatureVersion`: `1.0`
  - `SignatureNonce`: 随机字符串
  - `Timestamp`: ISO8601 格式时间戳
  - `SourceLanguage`: 源语言（ja/en）
  - `TargetLanguage`: 目标语言（zh）
  - `SourceText`: 待翻译文本

### 4.4 Google翻译实现
- 使用 `requests` 库调用 Google 翻译 API
- API 端点：`https://translate.googleapis.com/translate_a/single`
- 请求参数：
  - `client`: `gtx`
  - `sl`: 源语言（ja/en）
  - `tl`: 目标语言（zh）
  - `dt`: `t`（返回文本）
  - `q`: 待翻译文本
- **注意**：这是非官方 API，可能存在稳定性问题，用户需自行配置网络代理

## 5. 与OCR/翻译线程的集成

### 5.1 线程内流程
OCR/翻译线程的循环在成功识别文本后，调用文本去重模块判断是否需要翻译，若需要则调用翻译模块，最后通过信号发送结果。
```python
    OCR/翻译线程循环（简化）
    while running:
        task = task_queue.get()
        ocr_result = ocr_engine.recognize(task.image)
        if not ocr_result.success:
            continue

        # 文本去重
        if text_deduplicator.should_translate(ocr_result.text):
            trans_result = translator.translate(ocr_result.text, source_lang)
            if trans_result.success:
                self.signal_translation_completed.emit(
                    ocr_result.text, trans_result.translated_text
                )
            else:
                # 发送错误信号，悬浮窗显示错误提示
                self.signal_translation_failed.emit(trans_result.error)
```

### 5.2 信号定义
- `translation_completed(original_text, translated_text)`
- `translation_failed(error_message)`

主线程连接这些信号，更新悬浮窗显示。

## 6. 配置参数的使用

### 6.1 文本去重阈值
- 从配置读取 `ocr.text_threshold`，默认 75
- `config_helper.get("ocr", {}).get("text_threshold", 75)`

### 6.2 翻译配置
- 引擎类型：`config_helper.get("translation", {}).get("engine")`
- 阿里云密钥：
  - `config_helper.get("translation", {}).get("alibaba", {}).get("access_key_id")`
  - `config_helper.get("translation", {}).get("alibaba", {}).get("access_key_secret")`
- Google API Key：
  - `config_helper.get("translation", {}).get("google", {}).get("api_key")`

### 6.3 显示配置
- 字体：从配置读取 `display.font_family`、`display.font_size`，创建 `QFont` 应用于原文和译文标签。
- 颜色：从配置读取 `display.font_color`（译文颜色），原文颜色可固定为灰色（如 #AAAAAA）。
- 透明度：从配置读取 `display.bg_opacity`，用于绘制背景。

## 7. 异常处理

### 7.1 翻译API失败
- 在悬浮窗显示错误提示，原文可保留或清空。
- 连续失败记录日志，但不影响OCR循环。

### 7.2 网络错误处理
- 捕获 `requests.exceptions.RequestException` 及其子类
- 返回 `TranslationResult(success=False, translated_text="", error="网络错误")`

## 8. 与主线程的通信

- 使用 `pyqtSignal` 定义两个信号，由OCR/翻译线程发射，主线程（悬浮窗口所在线程）的槽函数接收。
- 确保信号连接类型为 `Qt.AutoConnection`（默认跨线程安全）。

## 9. 内存与性能

- 悬浮窗口只保留最新原文和译文，不存储历史。
- 翻译结果字符串可能较长，但视觉小说文本通常较短，无压力。
