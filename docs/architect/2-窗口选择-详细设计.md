# 窗口选择界面 - 详细技术设计文档

## 1. 界面概述
窗口选择界面是软件启动后的第二个界面，用户在此从系统当前所有可见窗口中选择需要监听的游戏或应用程序窗口。主要功能包括：
- 自动枚举并展示符合条件的窗口列表
- 提供刷新按钮重新扫描窗口
- 显示选中窗口的尺寸信息以供确认
- 通过“上一步”返回翻译设置界面，“下一步”进入截图区域选择界面

## 2. 界面实现技术要点

### 2.1 界面容器与布局
- 使用 `QWidget` 作为界面容器，采用 `QVBoxLayout` 垂直布局。
- 顶部放置引导文案 `QLabel`，样式可稍作突出。
- 中间区域使用 `QHBoxLayout` 放置下拉框和刷新按钮。
- 下方使用 `QGroupBox` 或 `QFrame` 作为窗口信息展示区，内部放置两个 `QLabel` 分别显示窗口尺寸和句柄。
- 底部使用 `QHBoxLayout` 放置“上一步”和“下一步”按钮。

### 2.2 窗口列表下拉框
- 控件：`QComboBox`
- 初始状态：设置占位文本“正在扫描窗口...”，同时启动窗口枚举线程。
- 枚举完成后，清空并填充所有符合条件窗口，若没有窗口则添加一项“未找到可用窗口”并禁用该项。
- 下拉项的数据存储：使用 `setItemData` 为每个项存储窗口句柄（HWND 作为整数或字符串），便于后续获取。
- 显示格式：`窗口标题 - 进程名`（例如“Summer Pockets - summer.exe”）。若标题为空，可用“无标题”替代。

### 2.3 刷新按钮
- 控件：`QPushButton`，设置图标 `QStyle::SP_BrowserReload` 或使用文本“刷新”。
- 点击后立即重新执行窗口枚举，更新下拉列表，同时清空之前的选择和信息展示区。
- 为避免短时间内多次点击，可在点击后暂时禁用按钮，枚举完成后再启用。

## 3. 窗口枚举模块设计

### 3.1 枚举条件
- 窗口必须可见（`IsWindowVisible` 为真）
- 窗口必须有标题（`GetWindowText` 返回非空字符串），或至少是有效的应用程序窗口
- 窗口非最小化（可通过 `IsIconic` 判断）
- **排除自身应用窗口**：获取当前进程ID，枚举窗口时过滤掉属于本进程的窗口
- 可选：只考虑顶层窗口（`GetParent` 为空）

### 3.2 枚举实现方式
- 使用 `win32gui.EnumWindows` 配合回调函数收集窗口信息。
- 回调中获取窗口标题、类名、进程ID（通过 `win32process.GetWindowThreadProcessId`）。
- 若窗口符合条件，记录句柄、标题、进程名（通过进程ID查询可执行文件名）。
- 为提升性能，可在工作线程中执行枚举，避免阻塞UI。

### 3.3 获取进程名
- 通过 `win32process.GetWindowThreadProcessId` 获取进程ID。
- 使用 `win32api.OpenProcess` 和 `psutil` 或 `win32process` 模块获取进程可执行文件名。
- 为简化，可使用 `psutil.Process(pid).name()`，但需引入 `psutil` 依赖；或使用 `win32api.GetModuleFileNameEx` 直接获取。

### 3.4 工作线程与UI更新
- 使用 `QThread` 派生一个工作线程类，执行窗口枚举。
- 枚举完成后通过信号传递窗口列表（结构体列表，每个包含 `hwnd, title, process_name`）给主线程。
- 主线程槽函数负责更新 `QComboBox` 和信息展示区。

## 4. 窗口信息展示区

### 4.1 信息获取时机
- 当用户在下拉框中选择一项时，触发 `currentIndexChanged` 信号。
- 从该项的存储数据中取出窗口句柄，调用 `win32gui.GetWindowRect` 获取窗口尺寸（`left, top, right, bottom`）。
- 更新信息展示区的 `QLabel` 显示宽度、高度和句柄值（格式化为十六进制）。

### 4.2 实时更新
- 若用户切换选项，信息区内容同步更新。
- 若点击刷新清空列表，信息区也清空。

## 5. 底部按钮控制

### 5.1 “下一步”按钮状态
- 初始状态：禁用（因下拉框无有效选中项）。
- 当用户选中一个有效窗口项（即不是“未找到可用窗口”这类占位项）时，启用“下一步”。
- 点击“下一步”时，需再次验证窗口是否仍然有效（例如调用 `IsWindow` 或尝试获取窗口矩形），若无效则弹出警告并阻止跳转。

### 5.2 “上一步”按钮
- 始终可点击。
- 点击后不保存当前窗口选择，直接返回翻译设置界面。

## 6. 数据保存与传递

### 6.1 配置存储
- 窗口句柄（HWND）应作为整数保存在全局配置对象中，字段名为 `window.hwnd`。
- 同时可保存窗口标题作为备注（用于显示），但不用于功能逻辑。
- 点击“下一步”时，调用配置管理器的 `set('window.hwnd', hwnd)`。

### 6.2 向后续界面传递
- 窗口选择界面不直接向下一界面传递数据，而是通过配置管理器存储，后续界面（区域选择）从配置读取窗口句柄。
- 区域选择界面打开时，通过配置获取当前窗口句柄，进行截图。

## 7. 异常与边界处理

### 7.1 窗口列表为空
- 枚举完成后若无窗口，下拉框添加一项“未找到可用窗口”，并设置为不可选。
- 信息展示区清空，“下一步”保持禁用。
- 提示用户检查目标应用是否打开且未最小化。

### 7.2 窗口在选中后关闭
- 点击“下一步”时，若窗口已无效，弹出警告框，提示用户重新选择。
- 可选择自动刷新窗口列表或让用户手动刷新。

### 7.3 枚举过程失败
- 若枚举线程抛出异常，应捕获并在UI上显示错误提示，下拉框显示“窗口扫描失败，请重试”，同时刷新按钮保持可用。

## 8. 与配置管理器的交互
- 界面打开时，从配置管理器读取已有的窗口句柄（如果有），并在下拉列表中尝试选中对应的项（通过句柄匹配）。
- 若无匹配项或句柄无效，则不选中任何项。
- 界面关闭（点击下一步）时，保存最新选中的句柄。

## 9. 性能考虑
- 每次刷新时重新枚举，线程应可中断（如正在枚举时用户再次点击刷新，应停止前一个线程）。
- 获取进程名可能较耗时，可考虑缓存进程名或只获取一次。
