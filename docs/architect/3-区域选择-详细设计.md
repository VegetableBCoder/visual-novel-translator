# 截图区域选择界面 - 详细技术设计文档

## 1. 设计概述

采用类似 Adobe Lightroom 裁剪工具的交互模式，通过半透明遮罩层与亮色选区的强烈视觉对比，实现直观的区域选择体验。核心功能包括：

- 显示目标窗口的实时截图，支持缩放适应窗口
- 提供遮罩层（60%透明度）与亮色选区的强对比视觉效果
- 支持 8 个控制点的选区创建、移动、拉伸操作
- 实时显示选区尺寸和相对位置信息
- 提供重新截屏按钮，更新截图
- 通过"上一步"返回窗口选择界面，"下一步"保存选区比例坐标

---

## 2. 视觉交互设计

### 2.1 遮罩效果

整个截图显示区域被划分为两个视觉层次：

#### 外部区域（遮罩层）
- 覆盖半透明黑色遮罩，透明度约 60%（`QColor(0, 0, 0, 153)`）
- 呈现暗化效果，降低视觉注意力
- 随选区变化实时调整遮罩边界

#### 内部区域（选区）
- 保持原始图像的完整亮度和色彩
- 作为用户关注的焦点区域
- 周围由 2 像素宽的亮蓝色边框界定（`QColor(0, 120, 215)`）

### 2.2 选区边框与控制点

选区边界线上分布 **8 个控制点**（手柄）：

| 位置 | 数量 | 形状 | 鼠标光标 | 功能 |
|------|------|------|----------|------|
| 四个角 | 4 | 6×6 像素正方形 | 斜向双向箭头 | 同时调整水平与垂直方向 |
| 四条边中点 | 4 | 6×6 像素正方形 | 水平/垂直箭头 | 调整对应方向 |

控制点视觉特征：
- 尺寸：6×6 像素
- 颜色：白色填充，边框与选区边框同色
- 鼠标悬停时：高亮显示，边框加粗或变色

## 3. 界面容器与布局

- 使用 `QWidget` 作为界面容器，采用 `QVBoxLayout` 垂直布局
- 顶部放置窗口标题 `QLabel`：**"选择识别区域"**
- 下方放置引导文案 `QLabel`：*"请在下方截图上框选需要OCR识别的文字区域"*
- 中间区域放置自定义的 `RegionSelectWidget`（继承自 `QWidget`），用于显示截图并处理选区交互
- 再下方放置水平布局，包含：
  - 左侧显示原始截图尺寸的 `QLabel`：格式 `原始截图: 1280x720`
  - 右侧"重新截屏"按钮 `QPushButton`
- 选区信息显示区，使用两个 `QLabel` 分别显示：
  - 选区尺寸：`宽度: 640px, 高度: 180px`
  - 相对位置：`左: 25.0%, 上: 70.0%, 右: 75.0%, 下: 85.0%`
- 底部使用 `QHBoxLayout` 放置"上一步"和"下一步"按钮

## 4. 自定义截图部件 RegionSelectWidget

### 4.1 功能职责
- 加载并显示截图（PIL Image 或 QPixmap）
- 处理图片缩放，保持宽高比，居中显示
- 处理所有鼠标事件，实现选区的创建、移动、拉伸
- 绘制半透明遮罩和选区边框及控制点
- 提供选区信息的实时更新

### 4.2 数据结构
```python
class RegionSelectWidget(QWidget):
    def __init__(self):
        super().__init__()
        # 图像相关
        self.original_pixmap = None    # 原始 QPixmap（未缩放）
        self.original_size = None      # 原始图片尺寸 (width, height)
        self.scaled_pixmap = None      # 缩放后的 QPixmap
        self.current_scale = 1.0       # 当前缩放因子
        self.pixmap_rect = QRectF()     # 图片在控件中的显示区域（控件坐标）

        # 选区相关
        self.selection_rect = QRectF() # 选区在图片显示区域中的坐标（相对于 pixmap_rect）
        self.min_selection_size = 30   # 选区最小尺寸（像素）

        # 交互状态
        self.mode = 'none'             # 当前操作模式：none, creating, moving, resizing
        self.resize_direction = None   # 拉伸方向：'left','right','top','bottom','topleft','topright','bottomleft','bottomright'
        self.drag_start_pos = None     # 拖拽起始点（控件坐标）
        self.original_rect = None      # 操作前的选区备份

        # 控制点相关
        self.handle_size = 6           # 控制点尺寸（像素）
        self.hovered_handle = None     # 当前悬停的控制点

        # 信号节流
        self._signal_timer = None      # QTimer，用于信号节流
```

### 4.3 图片加载与缩放

#### 设置图片
外部调用 `set_pixmap(qpixmap)` 设置新截图。

#### 缩放与居中逻辑
当窗口大小改变时，调用 `update_scale()` 方法：

1. 计算保持宽高比的缩放因子（取宽高比中较小者）
2. 根据缩放因子计算缩放后图片的实际像素尺寸
3. 将图片在窗口中居中放置（水平与垂直留出相等空白）
4. 更新 `pixmap_rect`（用于坐标转换）和 `current_scale`

#### 默认选区生成
首次加载图片后，若没有选区，自动生成默认选区：
- 宽度：图片显示区域宽度的 60%
- 高度：图片显示区域高度的 30%
- 位置：水平居中，垂直居中

## 5. 交互状态模型

### 5.1 操作模式分类

系统维护 5 种操作状态：

| 状态名称 | 触发条件 | 鼠标光标 | 行为描述 |
|----------|----------|----------|----------|
| `none` | 无操作 | 默认箭头 | 仅响应光标变化 |
| `creating` | 点击遮罩区域 | 十字准星 | 从点击处拖拽绘制新选区 |
| `moving` | 点击选区内部 | 四向箭头 | 整体平移选区位置 |
| `resizing` | 点击控制点 | 方向箭头 | 根据方向调整对应边界 |

### 5.2 鼠标事件处理

#### 5.2.1 鼠标按下 (mousePressEvent)

1. 获取鼠标在控件中的位置 `pos`
2. 判断 `pos` 是否在图片显示区域 `pixmap_rect` 内
3. 按优先级判断操作类型：
   - **控制点命中**：设置 `mode='resizing'`，记录拉伸方向和起始点，保存 `original_rect`
   - **选区内部命中**：设置 `mode='moving'`，记录起始点，保存 `original_rect`
   - **遮罩区域命中**：设置 `mode='creating'`，记录起始点，初始化 `selection_rect` 为极小矩形
4. 点击在图片区域外，忽略

#### 5.2.2 鼠标移动 (mouseMoveEvent)

根据当前 `mode` 进行不同处理：

- **creating**：更新 `selection_rect` 为从起始点到当前点的矩形，并应用边界约束
- **moving**：计算鼠标偏移量，将 `selection_rect` 平移相应量，并限制不超出图片边界
- **resizing**：根据拉伸方向，计算新的矩形边界，保持最小尺寸，并限制不超出图片边界
- **none**：根据鼠标位置更新光标形状和悬停控制点

每次更新 `selection_rect` 后：
- 调用 `update()` 触发重绘
- 通过节流定时器发送 `selectionChanged` 信号

#### 5.2.3 鼠标释放 (mouseReleaseEvent)

将 `mode` 重置为 `'none'`，结束操作。

若选区太小（宽度或高度小于最小像素），可自动调整或保持原样。

## 6. 坐标系统设计

### 6.1 四层坐标系

| 坐标系 | 定义 | 用途 |
|--------|------|------|
| **控件坐标** | 相对于窗口部件左上角 | 鼠标事件输入 |
| **图片显示坐标** | 相对于缩放后图片左上角 | 选区存储 |
| **原始图片坐标** | 相对于原始截图左上角 | 尺寸计算 |
| **相对比例坐标** | 0-1 的归一化值 | 最终存储 |

### 6.2 坐标转换方法

#### 控件坐标 → 图片显示坐标
```python
def widget_to_display(self, widget_pos: QPointF) -> QPointF:
    """将控件坐标转换为图片显示区域坐标"""
    return widget_pos - self.pixmap_rect.topLeft()
```

#### 图片显示坐标 → 原始图片坐标
```python
def display_to_original(self, display_pos: QPointF) -> QPointF:
    """将图片显示区域坐标转换为原始图片坐标"""
    return display_pos / self.current_scale
```

#### 原始图片坐标 → 相对比例坐标
```python
def original_to_relative(self, original_rect: QRectF) -> dict:
    """将原始图片坐标矩形转换为相对比例坐标（0-1）"""
    orig_w, orig_h = self.original_size
    return {
        "left": original_rect.left() / orig_w,
        "top": original_rect.top() / orig_h,
        "right": original_rect.right() / orig_w,
        "bottom": original_rect.bottom() / orig_h
    }
```

#### 获取当前选区的相对比例
```python
def get_selection_ratio(self) -> dict:
    """获取当前选区的相对比例坐标（0-1）"""
    # 1. 图片显示坐标 → 原始图片坐标
    orig_rect = QRectF(
        self.selection_rect.left() / self.current_scale,
        self.selection_rect.top() / self.current_scale,
        self.selection_rect.width() / self.current_scale,
        self.selection_rect.height() / self.current_scale
    )
    # 2. 原始图片坐标 → 相对比例坐标
    return self.original_to_relative(orig_rect)
```

## 7. 边界约束规则

### 7.1 边界约束

选区矩形必须满足：
- 左边界 ≥ 0
- 上边界 ≥ 0
- 右边界 ≤ 图片显示区域宽度
- 下边界 ≤ 图片显示区域高度

### 7.2 最小尺寸约束

选区矩形的最小尺寸限制：
- 最小宽度：30 像素（原始图片坐标）
- 最小高度：30 像素（原始图片坐标）

### 7.3 约束函数

统一约束函数用于所有选区操作：

```python
def constrain_rect(self, rect: QRectF) -> QRectF:
    """约束矩形在图片区域内且不小于最小尺寸"""
    # 约束边界
    rect.setLeft(max(rect.left(), 0))
    rect.setTop(max(rect.top(), 0))
    rect.setRight(min(rect.right(), self.pixmap_rect.width()))
    rect.setBottom(min(rect.bottom(), self.pixmap_rect.height()))

    # 约束最小尺寸（在边界约束后的基础上）
    if rect.width() < self.min_selection_size:
        if rect.left() == 0:
            rect.setRight(self.min_selection_size)
        else:
            rect.setLeft(rect.right() - self.min_selection_size)

    if rect.height() < self.min_selection_size:
        if rect.top() == 0:
            rect.setBottom(self.min_selection_size)
        else:
            rect.setTop(rect.bottom() - self.min_selection_size)

    return rect
```

## 8. 绘制渲染顺序

### 8.1 绘制事件 (paintEvent)

界面重绘时遵循以下绘制顺序：

1. **背景填充**
   - 使用中性色填充整个控件区域

2. **原始图像绘制**
   - 将缩放后的图片（`scaled_pixmap`）绘制在 `pixmap_rect` 位置

3. **遮罩层绘制**
   - 绘制图片显示区域左、右、上、下四个矩形区域
   - 使用半透明黑色填充（透明度 60%）
   - 选区矩形区域被遮罩排除（保持原样）

4. **选区边框绘制**
   - 沿选区矩形（`selection_rect`）绘制 2 像素宽的亮蓝色边框
   - 使用 `QPainter.setPen` 设置边框样式

5. **控制点绘制**
   - 在八个位置（四个角 + 四条边中点）绘制 6×6 像素控制点
   - 白色填充，蓝色边框
   - 鼠标悬停的控制点高亮显示

### 8.2 窗口大小变化 (resizeEvent)

重写 `resizeEvent` 方法：
- 调用 `update_scale()` 更新缩放因子和显示区域
- 如果选区存在，按比例 `current_scale` 调整选区矩形
- 调用 `update()` 触发重绘

## 9. 控制点检测

### 9.1 控制点位置计算

```python
def get_handle_rects(self) -> dict:
    """计算所有控制点的矩形区域（图片显示坐标）"""
    r = self.selection_rect
    hs = self.handle_size
    return {
        'topleft': QRectF(r.left() - hs/2, r.top() - hs/2, hs, hs),
        'top': QRectF(r.center().x() - hs/2, r.top() - hs/2, hs, hs),
        'topright': QRectF(r.right() - hs/2, r.top() - hs/2, hs, hs),
        'left': QRectF(r.left() - hs/2, r.center().y() - hs/2, hs, hs),
        'right': QRectF(r.right() - hs/2, r.center().y() - hs/2, hs, hs),
        'bottomleft': QRectF(r.left() - hs/2, r.bottom() - hs/2, hs, hs),
        'bottom': QRectF(r.center().x() - hs/2, r.bottom() - hs/2, hs, hs),
        'bottomright': QRectF(r.right() - hs/2, r.bottom() - hs/2, hs, hs)
    }
```

### 9.2 光标形状更新

根据鼠标位置设置光标形状：

- 在控制点上：根据方向设置对应的双向箭头光标
- 在选区内：`Qt.SizeAllCursor`（四向箭头）
- 在图片其他区域：`Qt.CrossCursor`（十字准星）
- 在图片外：`Qt.ArrowCursor`（默认箭头）

## 12. 主界面逻辑 (RegionSelectDialog)

### 12.1 初始化与截图加载

- 构造函数接收配置管理器引用
- 进入界面时，从配置管理器获取当前窗口句柄
- 调用 `CaptureService.capture_window(hwnd)` 获取 `PIL Image`
- 将 `PIL Image` 转换为 `QPixmap`：
  ```
  QPixmap.fromImage(
      QImage(image.tobytes(), image.width, image.height, QImage.Format_RGB888)
  )
  ```
- 若截图失败：弹出错误提示，禁用"下一步"，保留"重新截屏"和"上一步"

### 12.2 选区信息更新

连接 `RegionSelectWidget` 的 `selectionChanged` 信号到槽函数 `update_info_label`。

槽函数中：
- 获取当前选区（图片显示区域坐标）
- 转换为原始图片坐标（除以缩放因子）
- 计算尺寸和相对比例
- 更新两个信息标签

### 12.3 重新截屏

点击"重新截屏"按钮时：
- 重新调用截图模块获取新截图
- 更新 `RegionSelectWidget` 中的图片
- 选区重置为默认大小和位置

### 12.4 下一步

点击"下一步"时：
- 从 `RegionSelectWidget` 获取当前选区的相对比例坐标
- 调用 `ConfigManager.set('window.region', region_dict)`
- 保存配置
- 切换到运行参数配置界面

### 12.5 上一步

直接返回窗口选择界面，不保存选区。

## 13. 信号节流机制

为避免高频信号发送导致 UI 卡顿，使用 `QTimer` 实现信号节流：

```python
# 初始化
self._signal_timer = QTimer()
self._signal_timer.setSingleShot(True)
self._signal_timer.timeout.connect(self._emit_selection_changed)

# 在 mouseMoveEvent 中
def mouseMoveEvent(self, event):
    # ... 更新 selection_rect ...
    self._signal_timer.start(50)  # 50ms 后发送信号

def _emit_selection_changed(self):
    self.selectionChanged.emit(self.selection_rect)
```

## 14. 异常处理

### 截图失败
- 界面上方显示错误提示（`QMessageBox`` 或临时标签）
- 禁用"下一步"

- 提示用户返回上一步重新选择

- 捕获异常，显示错误并回退

## 16. 性能考虑

- 绘制操作在 `paintEvent` 中完成，Qt 自动进行增量更新
- 图片缩放使用 `Qt.SmoothTransformation` 保证质量
- 信号发送使用节流机制（50ms），避免高频更新
- 鼠标移动事件中保持轻量计算
- 缩放后的图片可缓存，减少重复计算
