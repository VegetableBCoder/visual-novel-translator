# 翻译结果悬浮窗 - 详细技术设计文档

## 1. 模块概述
翻译结果悬浮窗是一个独立的、始终置顶的、半透明背景的窗口，用于实时显示OCR识别出的原文和翻译后的译文。该窗口运行于主线程（UI线程），通过信号槽机制接收来自OCR/翻译线程的翻译结果，并根据用户在运行参数界面配置的字体、颜色、透明度等样式进行显示。窗口支持鼠标拖拽移动和边缘拖拽调整大小，并提供右键菜单（可选）以支持复制等操作。

## 2. 窗口类设计

### 2.1 类定义
- 类名：`FloatingTranslateWindow`
- 基类：`QWidget`
- 职责：
  - 管理窗口外观（半透明背景、无边框、置顶）。
  - 提供原文和译文的显示区域。
  - 处理鼠标事件实现拖动和大小调整。
  - 提供槽函数接收翻译结果并更新显示。
  - 应用配置管理器中的显示相关配置。
  - 保存/恢复窗口位置。

### 2.2 窗口标志与属性
- 设置窗口标志：
  - `Qt.FramelessWindowHint`：去除系统边框。
  - `Qt.WindowStaysOnTopHint`：始终置顶。
  - `Qt.Tool`：可选，使窗口不在任务栏显示图标，更像一个悬浮工具。
- 背景透明：使用 `setAttribute(Qt.WA_TranslucentBackground)` 启用窗口透明通道，然后在 `paintEvent` 中绘制半透明背景，确保文字不透明。

### 2.3 内部组件
- 使用 `QVBoxLayout` 作为主布局，边距可设小值（如2px）。
- 原文标签：`QLabel`，设置对象名 `originalLabel`，用于显示原文。
- 译文标签：`QLabel`，设置对象名 `translatedLabel`，用于显示译文。
- 两个标签均启用自动换行（`setWordWrap(True)`）。

## 3. 样式配置与更新

### 3.1 配置参数来源
- 从 `ConfigManager` 读取以下显示配置：
  - `display.font_family`：字体名称。
  - `display.font_size`：字体大小（译文），原文大小可设为译文大小的0.75倍。
  - `display.font_color`：译文颜色（如白色），原文颜色固定为灰色（如#AAAAAA）。
  - `display.bg_opacity`：背景透明度（0-100），0为完全透明，100为完全不透明。
  - `display.show_original`：是否显示原文（布尔值）。

### 3.2 应用样式
- 在窗口初始化时，从配置读取并设置：
  - 创建 `QFont` 对象，设置到两个标签。
  - 通过样式表或 `QPalette` 设置文字颜色。
  - 背景透明度通过 `paintEvent` 中的画刷实现（见第4节）。
- 当用户在运行参数界面修改配置并点击“应用”或“下一步”时，配置已保存。悬浮窗可通过以下两种方式之一更新样式：
  - 方式一：主窗口在配置保存后发送自定义信号（如 `configChanged`），悬浮窗连接该信号并在槽中重新读取配置并刷新样式。
  - 方式二：悬浮窗在每次显示新翻译结果时主动读取最新配置（简单但效率稍低）。推荐方式一，减少重复读取。

### 3.3 显示原文开关
- 若 `show_original` 为 `False`，原文标签可隐藏（`setVisible(False)`），布局自动调整。

## 4. 背景绘制与透明度

### 4.1 绘制机制
- 重写 `paintEvent` 方法。
- 使用 `QPainter` 绘制一个圆角矩形（可选）作为背景，填充颜色为黑色（或用户可配置），透明度根据 `bg_opacity` 计算（`alpha = int(255 * opacity / 100)`）。
- 背景矩形应覆盖整个窗口客户区。
- 绘制背景后，子控件（QLabel）会自动绘制在上面，由于子控件不透明，文字会正常显示。

### 4.2 实现要点
- 在构造函数中设置 `setAttribute(Qt.WA_TranslucentBackground)`，使窗口支持每像素透明度。
- 在 `paintEvent` 中，先绘制背景，然后调用父类 `paintEvent` 绘制子控件，或由Qt自动处理（因子控件是独立的窗口部件，需确保它们也被绘制）。通常只需绘制背景，子控件由Qt事件循环自动绘制。
- 注意：如果使用样式表为窗口设置背景色，可能覆盖 `paintEvent` 效果，因此应避免样式表设置背景，统一在 `paintEvent` 中绘制。

## 5. 鼠标事件与交互

### 5.1 窗口拖动
- 重写 `mousePressEvent`：记录鼠标按下位置（全局坐标）和窗口当前位置。
- 重写 `mouseMoveEvent`：若左键按下且处于拖动模式（非调整大小区域），计算偏移并调用 `move()` 移动窗口。
- 重写 `mouseReleaseEvent`：结束拖动。

### 5.2 调整大小
- 定义边缘检测区域：当鼠标在窗口边缘（如左、右、下、右下角等）时，改变光标形状。
- 在 `mousePressEvent` 中，根据光标位置判断是否进入调整大小模式，记录调整方向、窗口初始矩形和鼠标按下点。
- 在 `mouseMoveEvent` 中，根据调整方向计算新矩形，调用 `setGeometry` 更新窗口大小和位置。
- 调整大小时需考虑最小尺寸限制（如最小宽度200，最小高度100），避免窗口过小。

### 5.3 光标形状
- 在 `mouseMoveEvent` 中实时检测鼠标位置，设置对应光标：
  - 边缘左/右：`Qt.SizeHorCursor`
  - 边缘上/下：`Qt.SizeVerCursor`
  - 角落：`Qt.SizeFDiagCursor` / `Qt.SizeBDiagCursor`
  - 窗口内部（非边缘）：`Qt.ArrowCursor`（用于拖动）

### 5.4 右键菜单（可选）
- 重写 `contextMenuEvent`，创建 `QMenu`，添加动作：
  - “隐藏窗口”（或“显示/隐藏”，与托盘结合）
  - “退出程序”（或关闭悬浮窗）
- 动作触发时执行相应操作，如复制文本到剪贴板。

## 6. 数据流动与更新

### 6.1 接收翻译结果
- 在窗口类中定义槽函数：
```
slots:
void onTranslationCompleted(const QString &original, const QString &translated);
void onTranslationFailed(const QString &error);
```

- OCR/翻译线程通过信号发射结果，主线程（悬浮窗所在线程）的槽函数被调用。
- 信号连接应在主线程建立，确保跨线程安全（Qt默认自动处理）。

### 6.2 更新显示
- 在 `onTranslationCompleted` 中，根据 `show_original` 配置设置原文标签文本（可能隐藏），译文标签设置为译文。
- 若 `onTranslationFailed`，译文标签显示错误信息（如“翻译失败”），原文保留。
- 更新后，可调用 `adjustSize()` 或依赖布局自动调整窗口大小（若需要自动适应内容）。但考虑到用户可能手动调整过窗口大小，不应自动改变窗口尺寸，除非配置为自动调整。需求未要求自动调整，因此保持用户设定的大小。

### 6.3 清空显示
- 在暂停翻译或切换窗口时，可能需要清空显示内容。可提供 `clear()` 槽函数，由主窗口在适当时机调用。

## 7. 位置保存与恢复

### 7.1 保存位置
- 重写 `closeEvent`，在窗口关闭（不是退出程序）时，将当前窗口位置保存到配置管理器中：
```
config_manager.set("display.window_pos_x", self.x())
config_manager.set("display.window_pos_y", self.y())
config_manager.save()
```

- 注意：程序退出时也会触发 `closeEvent`，同样保存。

### 7.2 恢复位置
- 在窗口初始化时，读取配置中的位置，若存在且有效（在屏幕范围内），则调用 `move()` 移动到该位置；否则使用默认位置（如屏幕右下角）。
- 默认位置计算：获取屏幕尺寸，窗口尺寸，设置坐标为 `(screen_width - window_width - margin, screen_height - window_height - margin)`。

## 8. 与主窗口的联动

### 8.1 显示/隐藏控制
- 主窗口（或系统托盘）应能控制悬浮窗的显示和隐藏。比如暂停翻译时 这个窗口消失, 开始翻译时这个窗口出现
- 主窗口可连接相应按钮或快捷键信号到这些槽。

### 8.2 配置变更通知
- 主窗口在运行参数界面点击“应用”或“下一步”保存配置后，应发射一个自定义信号，如 `displayConfigChanged()`。
- 悬浮窗连接该信号，在槽中重新读取显示相关配置并刷新样式（字体、颜色、背景透明度）。

## 9. 异常处理

- 如果窗口初始化失败（如系统资源不足），应记录错误并尝试以默认样式创建。
- 当配置中的字体不存在时，`QFont` 会自动使用默认字体，无需特别处理。
- 若位置恢复时窗口部分超出屏幕，可自动调整到屏幕内（如调用 `qApp->primaryScreen()->availableGeometry()` 修正）。

## 10. 性能考虑

- 悬浮窗仅在接收到新翻译结果时更新文本，重绘频率低。
- 鼠标移动事件频繁，但处理逻辑简单，无性能问题。
- 背景绘制使用 `QPainter`，开销很小。
