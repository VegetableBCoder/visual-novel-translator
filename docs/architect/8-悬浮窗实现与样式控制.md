# 翻译结果悬浮窗 - 详细技术设计文档

## 1. 模块概述
翻译结果悬浮窗是一个独立的、始终置顶的、半透明背景的窗口，用于实时显示OCR识别出的原文和翻译后的译文。该窗口运行于主窗口的UI线程，通过信号槽机制接收来自OCR/翻译线程的翻译结果，并根据用户在运行参数界面配置的字体、颜色、透明度等样式进行显示。窗口支持鼠标拖拽移动和边缘拖拽调整大小，并提供两个图标按钮：复制按钮（复制翻译结果文本到剪贴板）和关闭按钮（隐藏悬浮窗，等同于暂停翻译）。

## 2. 窗口类设计

### 2.1 类定义
- 类名：`FloatingTranslateWindow`
- 基类：`QWidget`
- 职责：
  - 管理窗口外观（半透明背景、无边框、置顶）。
  - 提供原文和译文的显示区域。
  - 提供复制和关闭按钮。
  - 处理鼠标事件实现拖动和大小调整。
  - 提供槽函数接收翻译结果并更新显示。
  - 应用配置管理器中的显示相关配置。
  - 保存/恢复窗口位置。

### 2.2 窗口标志与属性
- 设置窗口标志：
  - `Qt.FramelessWindowHint`：去除系统边框。
  - `Qt.WindowStaysOnTopHint`：始终置顶。
  - `Qt.Tool`：可选，使窗口不在任务栏显示图标，更像一个悬浮工具。
- 背景透明：使用 `setAttribute(Qt.WA_TranslucentBackground)` 启用窗口透明通道，然后在 `paintEvent` 中绘制半透明背景，确保文字不透明。

### 2.3 内部组件
- 使用 `QVBoxLayout` 作为主布局，边距可设小值（如2px）。
- 原文标签：`QLabel`，设置对象名 `originalLabel`，用于显示原文。
- 译文标签：`QLabel`，设置对象名 `translatedLabel`，用于显示译文。
- 两个标签均启用自动换行（`setWordWrap(True)`）。
- 按钮容器：使用 `QHBoxLayout` 在窗口右上角放置按钮。
- 复制按钮：`QPushButton`，使用复制图标，点击时复制翻译结果文本。
- 关闭按钮：`QPushButton`，使用关闭图标（X），点击时隐藏悬浮窗（等同于暂停翻译）。

## 3. 样式配置与更新

### 3.1 配置参数来源
- 从 `ConfigManager` 读取以下显示配置：
  - `display.font_family`：字体名称。
  - `display.font_size`：字体大小（译文），原文大小固定为译文大小的0.75倍。
  - `display.font_color`：译文颜色（如白色）。
  - `display.original_font_color`：原文颜色，固定为灰色（#AAAAAA），这是设计决策以确保原文作为辅助信息不会喧宾夺主。
  - `display.bg_opacity`：背景透明度（0-100），0为完全透明，100为完全不透明。
  - `display.show_original`：是否显示原文（布尔值）。

### 3.2 应用样式
- 在窗口初始化时，从配置读取并设置：
  - 创建 `QFont` 对象，设置到两个标签。
  - 通过样式表或 `QPalette` 设置文字颜色。
  - 背景透明度通过 `paintEvent` 中的画刷实现（见第4节）。
- **样式更新机制**：由于翻译执行期间配置被锁定，用户无法在翻译过程中修改配置，因此悬浮窗无需处理动态配置更新。样式仅在窗口初始化时读取一次。

### 3.3 显示原文开关
- 若 `show_original` 为 `False`，原文标签使用 `setVisible(False)` 隐藏，布局自动收缩（QWidget的默认行为）。

## 4. 背景绘制与透明度

### 4.1 绘制机制
- 重写 `paintEvent` 方法。
- 使用 `QPainter` 绘制一个圆角矩形（可选）作为背景，填充颜色为黑色（或用户可配置），透明度根据 `bg_opacity` 计算（`alpha = int(255 * opacity / 100)`）。
- 背景矩形应覆盖整个窗口客户区。
- 绘制背景后，子控件（QLabel和按钮）会自动绘制在上面，由于子控件不透明，文字和按钮会正常显示。

### 4.2 实现要点
- 在构造函数中设置 `setAttribute(Qt.WA_TranslucentBackground)`，使窗口支持每像素透明度。
- 在 `paintEvent` 中，先绘制背景，然后调用父类 `paintEvent` 绘制子控件，或由Qt自动处理（因子控件是独立的窗口部件，需确保它们也被绘制）。通常只需绘制背景，子控件由Qt事件循环自动绘制。
- 注意：如果使用样式表为窗口设置背景色，可能覆盖 `paintEvent` 效果，因此应避免样式表设置背景，统一在 `paintEvent` 中绘制。

## 5. 鼠标事件与交互

### 5.1 窗口拖动
- 重写 `mousePressEvent`：记录鼠标按下位置（全局坐标）和窗口当前位置。
- 重写 `mouseMoveEvent`：若左键按下且处于拖动模式（非调整大小区域、非按钮区域），计算偏移并调用 `move()` 移动窗口。
- 重写 `mouseReleaseEvent`：结束拖动。
- **重要**：拖动不应与按钮点击冲突，在判断拖动前需检查鼠标是否在按钮区域内。

### 5.2 调整大小
- 定义边缘检测区域：当鼠标在窗口边缘（如左、右、下、右下角等）时，改变光标形状。
- 边缘检测阈值：**8像素**（即距离边缘8像素范围内视为边缘区域）。
- 在 `mousePressEvent` 中，根据光标位置判断是否进入调整大小模式，记录调整方向、窗口初始矩形和鼠标按下点。
- 在 `mouseMoveEvent` 中，根据调整方向计算新矩形，调用 `setGeometry` 更新窗口大小和位置。
- 调整大小时需考虑最小尺寸限制（如最小宽度200，最小高度100），避免窗口过小。

### 5.3 光标形状
- 在 `mouseMoveEvent` 中实时检测鼠标位置，设置对应光标：
  - 边缘左/右：`Qt.SizeHorCursor`
  - 边缘上/下：`Qt.SizeVerCursor`
  - 角落：`Qt.SizeFDiagCursor` / `Qt.SizeBDiagCursor`
  - 窗口内部（非边缘）：`Qt.ArrowCursor`（用于拖动）
  - 按钮上：按钮会自动设置光标，无需特别处理

## 6. 按钮功能实现

### 6.1 复制按钮
- 使用 Qt 标准图标：`QIcon.fromTheme("document-copy")` 或 `QIcon.fromTheme("edit-copy")`，若系统无此图标则使用自定义图标。
- 按钮样式：无边框、扁平化、半透明背景，鼠标悬停时背景变深。
- 点击行为：
  - 获取译文标签的文本内容。
  - 使用 `QApplication.clipboard().setText(text)` 复制到剪贴板。
  - 可选：显示短暂提示（如复制成功提示）。

### 6.2 关闭按钮
- 使用 Qt 标准图标：`QIcon.fromTheme("window-close")`，或使用简单的 "X" 图标。
- 按钮样式：无边框、扁平化、半透明背景，鼠标悬停时背景变深。
- 点击行为：
  - 发射自定义信号 `closeRequested()`。
  - 主窗口接收此信号后执行暂停翻译操作（等同于点击参数设置页的"暂停翻译"按钮）。
  - 悬浮窗隐藏：`self.hide()`。

### 6.3 按钮布局
- 使用 `QHBoxLayout` 将按钮放置在窗口右上角。
- 设置 `layout.setAlignment(Qt.AlignRight | Qt.AlignTop)` 使按钮靠右上角对齐。
- 使用 `layout.setContentsMargins()` 调整边距，使按钮不贴边。

## 7. 数据流动与更新

### 7.1 接收翻译结果
- 在窗口类中定义槽函数：
```python
@pyqtSlot(str, str)
def on_translation_completed(self, original: str, translated: str):

@pyqtSlot(str)
def on_translation_failed(self, error: str):
```

- OCR/翻译线程通过信号发射结果，主线程（悬浮窗所在线程）的槽函数被调用。
- 信号连接应在主线程建立，确保跨线程安全（Qt默认自动处理）。

### 7.2 更新显示
- 在 `on_translation_completed` 中，根据 `show_original` 配置设置原文标签文本（可能隐藏），译文标签设置为译文。
- 若 `on_translation_failed`，译文标签显示错误信息（如"翻译失败"），原文保留。
- 更新后，可调用 `adjustSize()` 或依赖布局自动调整窗口大小（若需要自动适应内容）。但考虑到用户可能手动调整过窗口大小，不应自动改变窗口尺寸，除非配置为自动调整。需求未要求自动调整，因此保持用户设定的大小。

### 7.3 清空显示
- 在暂停翻译或切换窗口时，可能需要清空显示内容。可提供 `clear()` 槽函数，由主窗口在适当时机调用。

## 8. 位置保存与恢复

### 8.1 保存位置
- 重写 `closeEvent`，在窗口关闭（不是退出程序）时，将当前窗口位置保存到配置管理器中：
```python
config_manager.set("display.window_pos_x", self.x())
config_manager.set("display.window_pos_y", self.y())
config_manager.set("display.window_width", self.width())
config_manager.set("display.window_height", self.height())
config_manager.save()
```
- 注意：程序退出时也会触发 `closeEvent`，同样保存。

### 8.2 恢复位置
- 在窗口初始化时，读取配置中的位置和大小，若存在且有效（在屏幕范围内），则调用 `setGeometry()` 设置；否则使用默认位置和大小。
- 默认位置计算：获取屏幕尺寸，窗口默认尺寸（如宽度400，高度200），设置坐标为 `(screen_width - window_width - margin, screen_height - window_height - margin)`，margin 可取 20 像素。
- 默认大小：宽度400像素，高度200像素。

## 9. 与主窗口的联动

### 9.1 显示/隐藏控制
- 主窗口应能控制悬浮窗的显示和隐藏：
  - 开始翻译时：调用悬浮窗的 `show()` 方法。
  - 暂停翻译时：调用悬浮窗的 `hide()` 方法。
- 主窗口连接悬浮窗的 `closeRequested` 信号到暂停翻译操作。

### 9.2 配置更新机制
- **说明**：由于翻译执行期间配置被锁定，用户无法在运行时修改配置，因此悬浮窗无需处理动态配置更新。
- 样式仅在以下两种情况下更新：
  1. 窗口初始化时从配置读取
  2. 暂停翻译后用户修改配置，再次开始翻译时窗口重新初始化（或重新加载样式）

## 10. 信号定义

悬浮窗定义以下信号供主窗口连接：

```python
# 关闭请求信号 - 用户点击关闭按钮时发射
closeRequested = pyqtSignal()
```

## 11. 异常处理

- 如果窗口初始化失败（如系统资源不足），应记录错误并尝试以默认样式创建。
- 当配置中的字体不存在时，`QFont` 会自动使用默认字体，无需特别处理。
- 若位置恢复时窗口部分超出屏幕，可自动调整到屏幕内（如调用 `QApplication.primaryScreen().availableGeometry()` 修正）。
- 复制操作失败时，静默处理（不显示错误提示），避免干扰用户。

## 12. 性能考虑

- 悬浮窗仅在接收到新翻译结果时更新文本，重绘频率低。
- 鼠标移动事件频繁，但处理逻辑简单，无性能问题。
- 背景绘制使用 `QPainter`，开销很小。
- 按钮点击事件频率低，无性能问题。
