# 截图服务与线程调度 - 详细技术设计文档

## 1. 概述
本文档描述视觉小说翻译器中截图服务及多线程调度的技术设计。截图服务负责根据用户配置定时捕获目标窗口指定区域的图像，并提供手动截图功能用于界面预览。线程调度模块管理后台截图线程和OCR/翻译线程，实现高效的异步处理，避免UI阻塞，并确保数据流的正确传递。

## 2. 截图服务设计

### 2.1 功能划分
截图服务需支持两种调用模式：
- **定时截图模式**：在自动翻译运行期间，由后台线程按固定间隔执行，用于持续监控游戏画面变化。
- **手动截图模式**：在界面操作中（如区域选择界面的“重新截屏”按钮）由主线程直接调用，用于获取当前窗口截图供用户预览。

### 2.2 截图服务接口
截图服务应封装为一个独立类 `CaptureService`，提供以下方法：

- `capture_window(hwnd) -> PIL.Image`：截取指定窗口的全屏图像。
- `capture_region(hwnd, region_ratio) -> PIL.Image`：根据相对比例区域截取窗口的指定区域（内部调用 `capture_window` 后再裁剪）。
- `get_window_rect(hwnd) -> tuple`：获取窗口当前矩形坐标，用于调试或信息显示。

### 2.3 实现要点
- 使用 `pywin32` 的 `PrintWindow` 方法（带 `PW_RENDERFULLCONTENT` 标志）截取游戏窗口，确保对硬件加速渲染有效。
- 截图操作可能因窗口最小化或关闭而失败，需捕获异常并返回 `None`。
- 手动截图模式无需额外处理，直接返回图像数据。

## 3. 线程调度设计

### 3.1 线程模型概述
程序启动后，在用户点击“开始翻译”时，启动以下后台线程：

| 线程 | 职责 | 与主线程交互 |
|------|------|-------------|
| **截图线程** | 定时执行截图、区域裁剪、图像哈希计算、图片级去重判断。当检测到画面变化时，将截图数据放入OCR任务队列。 | 无直接UI操作，通过信号或队列传递任务给OCR线程。 |
| **OCR/翻译线程** | 从任务队列中获取截图，执行OCR识别、文本去重、翻译API调用。翻译完成后通过信号将结果发送给主线程更新悬浮窗。 | 通过Qt信号将翻译结果发送给主线程，主线程更新UI。 |
| **主线程（UI线程）** | 处理用户界面事件，接收翻译结果更新悬浮窗。 | 负责启动/暂停/停止后台线程。 |

### 3.2 截图线程设计

#### 3.2.1 线程生命周期
- 由主线程在点击“开始翻译”时创建并启动。
- 点击“暂停翻译”时暂停循环（可通过原子标志控制），但线程不销毁，等待恢复。
- 点击“停止”或关闭程序时，通知线程退出并等待结束。

#### 3.2.2 工作循环
截图线程的核心是一个 `while` 循环，每次迭代执行：

1. 检查暂停标志，若暂停则休眠一小段时间后继续检查。
2. 从配置管理器读取当前窗口句柄、区域比例、截图间隔、图像相似度阈值。
3. 调用 `CaptureService.capture_region` 获取区域截图。
4. 若截图失败（窗口无效），则记录错误并休眠，等待下一次尝试。
5. 计算截图图像的 pHash 值（使用 `imagehash.phash`）。
6. 与上一次保存的 pHash 比较，计算汉明距离。
7. 若距离超过阈值（或上一次哈希为空），则判定画面变化：
   - 将截图图像和相关信息封装为任务对象，放入任务队列。
   - 更新“上一次哈希”为当前哈希。
8. 根据配置的截图间隔休眠。

#### 3.2.3 线程安全
- 配置读取：每次循环从配置管理器读取最新配置（配置管理器内部加锁）。
- 任务队列：使用线程安全的 `queue.Queue` 或 `QQueue`（配合QMutex）。
- 暂停/停止标志：使用 `QAtomicInt` 或 `threading.Event`。

### 3.3 OCR/翻译线程设计

#### 3.3.1 线程职责
- 从任务队列中取出任务（包含截图图像）。
- 调用 OCR 服务识别文本。
- 对识别出的文本进行文本级去重（与上一次翻译的文本比较）。
- 若文本有变化，调用翻译服务获取译文。
- 通过信号将原文和译文发送给主线程。

#### 3.3.2 工作循环
OCR/翻译线程同样运行一个循环，从队列中取任务，但为了避免忙等，可使用阻塞队列（如 `queue.Queue` 的 `get` 方法阻塞）。

1. 从队列获取任务（超时机制，以便检查停止标志）。
2. 调用 OCR 引擎的识别方法，获取文本和置信度。
3. 若置信度过低，可丢弃（可选，暂不实现）。
4. 与上一次成功翻译的文本进行 Levenshtein 相似度比较。
5. 若相似度低于阈值，则：
   - 调用翻译 API 获取译文。
   - 通过信号发射翻译结果（原文、译文、时间戳）。
   - 更新“上一次文本”为当前原文。
6. 否则，丢弃。

#### 3.3.3 异步 API 调用
翻译 API 调用可能耗时，直接在 OCR/翻译线程中同步调用即可，因为该线程本身就是用于处理耗时操作的。若担心单个任务阻塞队列，可考虑使用线程池，但当前设计单线程足够。

### 3.4 线程间通信

#### 3.4.1 截图线程 → OCR/翻译线程
- 使用线程安全的队列 `task_queue` 传递任务对象。
- 任务对象包含：截图图像（PIL Image）、时间戳。

#### 3.4.2 OCR/翻译线程 → 主线程
- 使用 Qt 的信号/槽机制（因为主线程运行 Qt 事件循环）。
- 定义信号 `translationCompleted(original_text, translated_text)`，在 OCR/翻译线程中发射，主线程的槽函数更新悬浮窗。
- Qt 信号是线程安全的，可以直接跨线程连接。

#### 3.4.3 主线程 → 后台线程
- 通过调用线程对象的公共方法（如 `pause()`、`stop()`）修改标志位。
- 需要加锁保护标志位。

### 3.5 状态管理

#### 3.5.1 运行状态
程序运行状态包括：空闲、运行中、暂停。由主线程的 `StateManager` 维护。

- **空闲**：未开始翻译，后台线程未启动。
- **运行中**：截图线程和 OCR/翻译线程均在运行。
- **暂停**：截图线程循环跳过截图操作（但线程不退出），OCR/翻译线程继续处理完队列中已有任务后暂停新任务处理（或直接暂停）。

#### 3.5.2 控制接口
- `start()`：创建并启动两个线程。
- `pause()`：设置暂停标志；截图线程暂停截图，OCR/翻译线程完成当前任务后暂停取新任务。
- `resume()`：清除暂停标志。
- `stop()`：设置停止标志，等待线程结束。

## 4. 数据流与任务队列

### 4.1 任务对象定义
```python
class OCRTask:
    def __init__(self, image, timestamp):
        self.image = image      # PIL Image
        self.timestamp = timestamp

```

### 4.2 队列容量

为防止 OCR / 翻译线程处理不过来导致队列无限增长，可设置队列最大长度。当队列满时，截图线程丢弃新的任务（即跳过本次变化）。

但考虑到视觉小说对话速度，一般不会出现明显积压。

### 4.3 去重状态共享

**图片级去重**

截图线程维护 `last_image_hash`，仅在线程内部使用，无需加锁。

**文本级去重**

OCR / 翻译线程维护 `last_translated_text`，仅在线程内部使用。

## 5. 与配置管理器的交互

截图线程每次循环需读取：

- 窗口句柄
- 区域比例
- 截图间隔
- 图像相似度阈值

OCR / 翻译线程需读取：

- 源语言
- 翻译引擎类型
- API 密钥
- 文本去重阈值

配置管理器应提供**线程安全的读取方法**（通过锁保护）。

由于读取频繁但配置变化不频繁，可考虑：

- 在每次循环开始时一次性读取所需配置快照
- 或使用读写锁（Read-Write Lock）

---

## 6. 异常处理

**截图失败**

若连续多次失败（如窗口关闭），截图线程应通知主线程（通过信号）弹出提示，并自动暂停。

**OCR 失败**

OCR 识别失败（返回空或置信度过低）时，应记录日志，不触发翻译。

**翻译 API 失败**

网络错误或密钥无效时：

- OCR / 翻译线程应捕获异常
- 通过信号发送错误信息
- 主线程可在悬浮窗显示“翻译失败”提示

**线程崩溃**

应捕获未处理异常，记录日志，并尝试安全退出。

---

## 7. 性能考虑

- 截图线程循环间隔不应低于 **100 ms**，避免过度消耗 CPU。
- 图像哈希计算（`pHash`）应使用缩略图（如 `32×32`）以提高速度，`imagehash.phash` 内部通常已做处理。
- OCR 和翻译操作是主要耗时点，应保证有足够的队列缓冲。

## 8. 时序图（文字描述）
```
用户点击“开始翻译”：
主线程 -> 创建并启动截图线程, OCR/翻译线程
截图线程循环：
    休眠间隔时间
    截图 -> 裁剪 -> 计算哈希 -> 与上次比较
    若变化 -> 将图像放入任务队列
OCR/翻译线程循环：
    从队列取任务
    OCR识别 -> 文本去重 -> 若变化 -> 翻译API
    发射翻译结果信号
主线程槽函数：
    接收信号 -> 更新悬浮窗文本
用户点击“暂停翻译”：
主线程 -> 设置暂停标志
截图线程检查暂停标志，跳过截图操作
OCR/翻译线程处理完当前任务后暂停
用户点击“恢复翻译”：
主线程 -> 清除暂停标志
线程恢复正常工作
用户关闭程序：
主线程 -> 设置停止标志，等待线程结束
```

## 9. 与UI模块的接口
* 主线程通过调用线程控制方法管理生命周期。
* 翻译结果信号定义在某个可访问的对象上（如主窗口或专门的翻译控制器）。
