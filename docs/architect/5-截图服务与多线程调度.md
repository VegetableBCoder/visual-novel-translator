# 截图服务与线程调度 - 详细技术设计文档

## 1. 概述
本文档描述视觉小说翻译器中截图服务及多线程调度的技术设计。截图服务负责根据用户配置定时捕获目标窗口的图像，然后根据选定的区域进行裁剪。线程调度模块管理后台截图+OCR线程和翻译线程，实现高效的异步处理，避免UI阻塞，并确保数据流的正确传递。

**设计原则：**
- 复用已有的 `WindowCapture` 类作为底层截图实现
- 区域裁剪逻辑独立封装在 `ImageProcessor` 中
- 统一使用 Qt 信号/槽机制进行线程间通信
- 配置锁定期间无需考虑线程安全问题
- 使用 OpenCV 图像差分代替 pHash（视觉小说对话框内容变化小，差分更敏感）

## 2. 截图服务设计

### 2.1 功能划分
截图服务需支持两种调用模式：
- **定时截图模式**：在自动翻译运行期间，由后台线程按固定间隔执行，用于持续监控游戏画面变化。
- **手动截图模式**：在界面操作中（如区域选择界面的"重新截屏"按钮）由主线程直接调用，用于获取当前窗口截图供用户预览。

### 2.2 底层截图实现
**复用 `src/core/window_capture.py` 中的 `WindowCapture` 类**

该类已经实现了以下功能：
- 使用 `pywin32` 的 `PrintWindow` 方法截取游戏窗口
- 支持 DPI 缩放处理
- 返回整个窗口的 PIL Image 对象

接口方法：
```python

    def capture_window(hwnd: int) -> Optional[Image.Image]

```

### 2.3 图像处理与区域裁剪
**新增 `src/service/image_processor.py` 模块**

该模块负责：
1. 根据相对比例坐标裁剪图像
2. 图像预处理（缩小、统一宽度等）

### 2.4 实现要点
- 截图操作可能因窗口最小化或关闭而失败，需捕获异常并返回 `None`。
- 区域裁剪使用相对比例坐标转换为绝对像素坐标。
- 图像缩小使用 Lanczos 或 bicubic 插值以保持质量。

## 3. OCR服务设计

### 3.1 功能划分
OCR服务负责：
1. **图像差分去重判断**：使用 OpenCV 比较当前图像与上一次图像的差分值
2. **OCR识别**：调用 PaddleOCR 识别图像中的文本
3. **发送结果**：通过信号发送识别结果给翻译线程

### 3.2 图像差分去重原理

**为什么使用差分而不是pHash？**
- 视觉小说游戏中，对话框内容变化通常很小（如闪烁、光标移动）
- pHash 对整体结构敏感，可能忽略细微变化
- 图像差分直接比较像素差异，对文本变化更敏感

**差分流程：**
1. 将图像转换为灰度
2. 缩小图像以提高性能
3. 计算两帧图像的绝对差分
4. 统计差分图像中变化像素的数量或平均差分值
5. 与阈值比较，判断是否发生显著变化

### 3.3 OCR服务接口
**新增 `src/service/ocr_service.py` 模块**

```python
import cv2
import numpy as np
from PyQt5.QtCore import QObject, pyqtSignal
from typing import Optional
import time

class OCRService(QObject):
    """OCR服务 - 图像差分去重和OCR识别"""

    # 信号定义
    ocr_result = pyqtSignal(str, float)  # 识别结果文本, 时间戳
    error_signal = pyqtSignal(str)  # 错误信息

    def __init__(self, parent=None):
        super().__init__(parent)

        # 上一次图像（用于差分比较）
        self._last_image: Optional[np.ndarray] = None

        # OCR引擎（依赖注入）
        self._ocr_engine = None
        self._image_processor = None

        # 配置快照
        self._config_snapshot = None

    def set_dependencies(self, ocr_engine, image_processor):
        """设置依赖对象"""
        self._ocr_engine = ocr_engine
        self._image_processor = image_processor

    def set_config_snapshot(self, config: dict):
        """设置配置快照"""
        self._config_snapshot = config

    def process_image(self, image: Image.Image):
        """
        处理图像（差分去重 + OCR识别）

        Args:
            image: 裁剪后的图像
        """
        if not self._config_snapshot:
            return

        # 获取配置参数
        image_threshold = self._config_snapshot.get("ocr.image_threshold", 10)
        resize_width = self._config_snapshot.get("ocr.resize_width", 320)
        resize_height = self._config_snapshot.get("ocr.resize_height", 240)

        try:
            # 转换为numpy数组
            image_array = np.array(image)

            # 转换为灰度
            gray = cv2.cvtColor(image_array, cv2.COLOR_RGB2GRAY)

            # 缩小图像以提高性能
            resized = cv2.resize(gray, (resize_width, resize_height), interpolation=cv2.INTER_AREA)

            # 首次图片，保存但不进行OCR
            if self._last_image is None:
                self._last_image = resized
                return

            # 计算差分
            diff = cv2.absdiff(self._last_image, resized)

            # 统计变化（阈值判断）
            _, thresholded = cv2.threshold(diff, 30, 255, cv2.THRESH_BINARY)
            changed_pixels = cv2.countNonZero(thresholded)

            # 计算变化比例
            total_pixels = resized.shape[0] * resized.shape[1]
            change_ratio = changed_pixels / total_pixels

            # 判断是否触发OCR（阈值越高，触发条件越严格）
            threshold_ratio = image_threshold / 100.0  # 转换为0-1

            if change_ratio < threshold_ratio:
                # 变化不大，跳过
                return

            # 更新上一次图像
            self._last_image = resized

            # 执行OCR识别
            result = self._ocr_engine.recognize(image)

            if result and "text" in result:
                text = result["text"]
                if text.strip():
                    self.ocr_result.emit(text, time.time())

        except Exception as e:
            self.error_signal.emit(f"OCR处理异常: {str(e)}")

    def reset(self):
        """重置去重状态（首次启动或区域改变时调用）"""
        self._last_image = None
```

## 4. 线程调度设计

### 4.1 线程模型概述
程序启动后，在用户点击"开始翻译"时，启动以下后台线程：

| 线程 | 职责 | 与主线程交互 |
|------|------|-------------|
| **截图+OCR线程** | 定时执行截图、区域裁剪。将裁剪后的图像传递给OCR服务进行差分去重和OCR识别。 | 通过信号将OCR识别结果发送给翻译线程。 |
| **翻译线程** | 接收OCR识别结果，进行文本级去重判断，调用翻译API，通过信号将翻译结果发送给主线程。积压时丢弃较早的识别结果。 | 通过Qt信号将翻译结果发送给主线程，主线程更新悬浮窗。 |
| **主线程（UI线程）** | 处理用户界面事件，接收翻译结果更新悬浮窗。 | 负责启动/暂停/停止后台线程。 |

**线程状态图：**
```
[主线程]
    ├─ start() → 创建并启动工作线程
    ├─ pause() → 设置暂停标志
    ├─ resume() → 清除暂停标志
    └─ stop() → 设置停止标志，等待线程结束

[截图+OCR线程]
    └─ while running:
        if paused: sleep & continue
        截图 → 裁剪 → 调用OCR服务处理

[OCRService]
    └─ process_image():
        图像差分 → 去重判断
        if 变化: OCR识别 → 发射信号

[翻译线程]
    └─ on_ocr_result(text, timestamp):
        if 有翻译任务在执行: 丢弃新的识别结果
        文本级去重
        if 文本变化: 翻译API → 发射信号给主线程
```

### 4.2 职责划分总结

| 组件 | 职责 |
|------|------|
| **截图Service (WindowCapture)** | 仅负责窗口截图，不做任何去重判断 |
| **ImageProcessor** | 负责区域裁剪和图像预处理（缩小） |
| **OCRService** | 1. 图像差分去重判断<br>2. OCR识别<br>3. 发送识别结果 |
| **翻译Service (TranslationThread)** | 1. 文本去重判断（Levenshtein）<br>2. 调用翻译API<br>3. 通知UI更新<br>4. 积压时丢弃较早结果 |

### 4.3 截图+OCR线程设计

**文件：`src/service/capture_thread.py`**

```python
from PyQt5.QtCore import QThread, pyqtSignal, QMutex, QWaitCondition
from PIL import Image
from typing import Optional

class CaptureThread(QThread):
    """截图线程 - 定时截图和区域裁剪"""

    # 信号定义
    error_signal = pyqtSignal(str)  # 错误信息

    def __init__(self, parent=None):
        super().__init__(parent)
        self._mutex = QMutex()
        self._wait_condition = QWaitCondition()

        # 配置快照
        self._config_snapshot = None

        # 状态标志
        self._running = False
        self._paused = False

        # 依赖注入
        self._window_capture = None
        self._image_processor = None
        self._ocr_service = None

    def set_dependencies(self, window_capture, image_processor, ocr_service):
        """设置依赖的服务对象"""
        self._window_capture = window_capture
        self._image_processor = image_processor
        self._ocr_service = ocr_service

    def set_config_snapshot(self, config: dict):
        """设置配置快照"""
        with QMutexLocker(self._mutex):
            self._config_snapshot = config
            # 重置OCR服务的去重状态
            if self._ocr_service:
                self._ocr_service.set_config_snapshot(config)
                self._ocr_service.reset()

    def start_capture(self):
        """开始截图"""
        self._paused = False
        if not self.isRunning():
            self._running = True
            self.start()

    def pause(self):
        """暂停截图"""
        with QMutexLocker(self._mutex):
            self._paused = True

    def resume(self):
        """恢复截图"""
        with QMutexLocker(self._mutex):
            self._paused = False
            self._wait_condition.wakeAll()

    def stop(self):
        """停止线程"""
"""
        with QMutexLocker(self._mutex):
            self._running = False
            self._paused = False
            self._wait_condition.wakeAll()

    def run(self):
        """线程主循环"""
        while True:
            # 检查停止标志
            with QMutexLocker(self._mutex):
                if not self._running:
                    break

                # 检查暂停标志
                if self._paused:
                    self._wait_condition.wait(self._mutex)
                    continue

            # 读取配置快照
            config = self._config_snapshot
            if not config:
                self.msleep(100)
                continue

            # 获取配置参数
            hwnd = config.get("window.hwnd")
            region = config.get("window.region", {})
            interval_ms = config.get("ocr.interval_ms", 1000)

            # 截图
            try:
                full_image = self._window_capture.capture_window(hwnd)
                if full_image is None:
                    self.msleep(interval_ms)
                    continue

                # 裁剪区域
                window_size = (full_image.width, full_image.height)
                cropped_image = self._image_processor.crop_region(
                    full_image, region, window_size
                )
                if cropped_image is None:
                    self.msleep(interval_ms)
                    continue

                # 调用OCR服务处理（包含差分去重和OCR识别）
                self._ocr_service.process_image(cropped_image)

            except Exception as e:
                self.error_signal.emit(f"截图线程异常: {str(e)}")

            # 休眠指定间隔
            self.msleep(interval_ms)
```

### 4.4 翻译线程设计

**文件：`src/service/translation_thread.py`**

```python
from PyQt5.QtCore import QObject, pyqtSignal
from typing import Optional

class TranslationThread(QObject):
    """翻译线程 - 文本去重和翻译"""

    # 信号定义
    translation_result = pyqtSignal(str, str, float)  # 原文, 译文, 时间戳
    error_signal = pyqtSignal(str)  # 错误信息

    def __init__(self, parent=None):
        super().__init__(parent)

        # 配置快照
        self._config_snapshot = None

        # 去重状态
        self._last_translated_text = ""

        # 翻译状态
        self._is_translating = False  # 是否正在翻译

        # 依赖注入
        self._translator = None
        self._text_deduplicator = None

    def set_dependencies(self, translator, text_deduplicator):
        """设置依赖的服务对象"""
        self._translator = translator
        self._text_deduplicator = text_deduplicator

    def set_config_snapshot(self, config: dict):
        """设置配置快照"""
        self._config_snapshot = config

    def on_ocr_result(self, text: str, timestamp: float):
        """槽函数 - 接收OCR识别结果"""
        if not text.strip():
            return

        # 如果正在翻译，丢弃新的识别结果
        if self._is_translating:
            return

        # 读取配置
        config = self._config_snapshot
        if not config:
            return

        # 文本级去重
        if self._text_deduplicator.is_similar(text, self._last_translated_text):
            return

        # 更新上一次文本
        self._last_translated_text = text

        # 标记正在翻译
        self._is_translating = True

        # 调用翻译API
        try:
            source_lang = config.get("language.source", "ja")
            target_lang = config.get("language.target", "zh")

            translated = self._translator.translate(text, source_lang, target_lang)

            if translated:
                self.translation_result.emit(text, translated, timestamp)

        except Exception as e:
            self.error_signal.emit(f"翻译失败: {str(e)}")
        finally:
            # 翻译完成，标记状态
            self._is_translating = False
```

### 4.5 线程间通信

#### 4.5.1 截图+OCR线程 → 翻译线程
- 使用 Qt 信号/槽机制连接
- 信号路径：`OCRService.ocr_result` → `TranslationThread.on_ocr_result`

#### 4.5.2 翻译线程 → 主线程
- 使用 Qt 信号/槽机制连接
- 信号：`TranslationThread.translation_result` → 主窗口槽函数

#### 4.5.3 主线程 → 工作线程
- 通过调用线程对象的公共方法控制生命周期
- 通过 `set_config_snapshot()` 传递配置快照

### 4.6 状态管理

#### 4.6.1 运行状态
- **IDLE**：未开始翻译，后台线程未启动
- **RUNNING**：截图+OCR线程和翻译线程均在运行
- **PAUSED**：截图线程暂停，翻译线程可继续处理已有任务

#### 4.6.2 配置锁定机制
- **翻译启动后**：配置参数锁定，界面控件变为只读
- **暂停翻译后**：配置参数解锁，界面控件恢复可编辑
- **恢复翻译前**：先保存新配置，然后传递配置快照

```python
from PyQt5.QtCore import QObject, pyqtSignal

class Scheduler(QObject):
    """任务调度器 - 管理截图+OCR线程和翻译线程线程"""

    # 信号定义（转发给主线程）
    translation_result = pyqtSignal(str, str, float)  # 原文, 译文, 时间戳
    error_signal = pyqtSignal(str)  # 错误信息

    def __init__(self, parent=None):
        super().__init__(parent)

        # 创建工作线程
        self._capture_thread = CaptureThread()
        self._ocr_service = OCRService()
        self._translation_thread = TranslationThread()

        # 连接信号
        self._connect_signals()

        # 状态
        self._config_snapshot = None

    def _connect_signals(self):
        """连接内部线程信号"""
        # OCR服务 → 翻译线程
        self._ocr_service.ocr_result.connect(
            self._translation_thread.on_ocr_result
        )

        # 错误信号转发
        self._capture_thread.error_signal.connect(self.error_signal)
        self._ocr_service.error_signal.connect(self.error_signal)
        self._translation_thread.error_signal.connect(self.error_signal)

        # 翻译线程 → 主线程
        self._translation_thread.translation_result.connect(
            self.translation_result
        )

    def set_dependencies(self, window_capture, image_processor, ocr_engine,
                      translator, text_deduplicator):
        """设置所有依赖的服务对象"""
        self._capture_thread.set_dependencies(
            window_capture, image_processor, self._ocr_service
        )
        self._ocr_service.set_dependencies(ocr_engine, image_processor)
        self._translation_thread.set_dependencies(
            translator, text_deduplicator
        )

    def set_config_snapshot(self, config: dict):
        """设置配置快照"""
        self._config_snapshot = config.copy()
        self._capture_thread.set_config_snapshot(self._config_snapshot)
        self._ocr_service.set_config_snapshot(self._config_snapshot)
        self._translation_thread.set_config_snapshot(self._config_snapshot)

    def start(self):
        """启动翻译"""
        self._capture_thread.start_capture()

    def pause(self):
        """暂停翻译"""
        self._capture_thread.pause()

    def resume(self):
        """恢复翻译"""
        self._capture_thread.resume()

    def stop(self):
        """停止翻译"""
        self._capture_thread.stop()
        self._capture_thread.wait()
```

### 5.2 去重状态

**图片级去重**
- 位置：OCRService内部
- 变量：`self._last_image` (numpy数组)
- 去重方式：OpenCV图像差分
- 无需加锁，仅线程内部访问

**文本级去重**
- 位置：翻译线程内部
- 变量：`self._last_translated_text`
- 去重方式：Levenshtein编辑距离
- 无需加锁，仅线程内部访问

## 6. 与配置管理器的交互

### 6.1 配置读取时机
- **启动翻译时**：从 `ConfigManager` 读取完整配置，创建配置快照
- **暂停后恢复时**：重新读取配置，更新配置快照

### 6.2 配置锁定机制
- **运行中**：配置界面参数锁定，`ConfigManager` 不会被修改
- **暂停后**：配置界面解锁，用户可修改参数
- **恢复前**：必须调用 `save_config()` 保存，然后传递新配置快照

### 6.3 线程安全说明
由于采用配置快照 + 配置锁定机制：
- 工作线程只读取配置快照，不修改
- 配置快照仅在主线程更新（启动/恢复时）
- **无需考虑配置访问的线程安全问题**

## 7. 异常处理

### 7.1 截图失败
- 窗口最小化或关闭时，`capture_window` 返回 `None`
- 记录日志，跳过本次循环

### 7.2 OCR 失败
- OCR 识别失败（返回空或异常）时，记录日志
- 通过 `error_signal` 发送错误信息

### 7.3 翻译 API 失败
- 网络错误或密钥无效时，捕获异常
- 通过 `error_signal` 发送错误信息
- 主线程可在悬浮窗显示"翻译失败"提示

### 7.4 线程崩溃
- 线程内部捕获未处理异常
- 通过 `error_signal` 通知主线程
- 记录完整错误日志

## 8. 性能考虑

### 8.1 截图间隔
- 最小间隔：100 ms（避免过度消耗 CPU）
- 推荐间隔：1000 ms（视觉小说文本变化通常较慢）

### 8.2 图像差分性能
- 图像缩小到 320x240 或更小，以提高差分速度
- 使用 OpenCV 的 INTER_AREA 插值进行缩小
- 差分计算是整数运算，非常快

### 8.3 OCR 性能
- OCR 是主要耗时操作（可能需要 100-500ms）
- 通过图像差分去重减少不必要的 OCR 调用

### 8.4 翻译性能
- 翻译 API 调用是网络操作，可能需要 1-3 秒
- 通过文本去重减少不必要的 API 调用
- 翻译线程中设置 `_is_translating` 标志，积压时丢弃新结果

### 9.1 启动流程
```
用户点击"开始翻译"
    ↓
run_config._start_translation()
    ├─ save_config() 保存配置
    ├─ scheduler.set_config_snapshot(config) 传递配置快照
    │    ├─ ocr_service.reset() 重置去重状态
    │    └─ 设置各组件的配置快照
    ├─ scheduler.start() 启动工作线程
    │    └─ capture_thread.start_capture()
    └─ start_signal.emit() 通知主窗口

capture_thread.run() 开始循环
    while running:
        ├─ capture_window(hwnd) 截图
        ├─ crop_region() 裁剪
        └─ ocr_service.process_image(cropped_image)

ocr_service.process_image()
    ├─ 图像差分计算
    ├─ 去重判断
    └─ if 变化:
        ├─ OCR识别
        └─ ocr_result.emit(text, timestamp)

translation_thread.on_ocr_result(text, timestamp)
    ├─ 检查 _is_translating 标志
    │    └─ if 正在翻译: 丢弃
    ├─ 文本级去重判断
    └─ if 新文本:
        ├─ 设置 _is_translating = True
        ├─ translate(text) 调用翻译API
        ├─ translation_result.emit(original, translated, timestamp)
        └─ 设置 _is_translating = False
```

## 12. 附录：图像差分参数说明

### 12.1 图像缩放参数

| 参数 | 默认值 | 说明 | 调节建议 |
|------|--------|------|----------|
| resize_width | 320 | 差分时图像宽度 | 较小值提高性能，较大值提高精度 |
| resize_height | 240 | 差分时图像高度 | 同上 |

### 12.2 差分阈值参数

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| image_threshold | 0-100 | 10 | 变化像素比例阈值，值越高触发条件越严格 |

**阈值调节建议：**
- **低阈值（5-10）**：对微小变化敏感，可能误触发
- **中阈值（10-30）**：推荐范围，适合大多数场景
- **高阈值（30-50）**：只对明显变化触发，适合快速对话场景

### 12.3 差分阈值换算

UI中的阈值（0-100）转换为差分比例（0.0-1.0）：
```python
threshold_ratio = image_threshold / 100.0
```

例如：
- 阈值 10 → 阈值比例 0.10（10%像素变化触发）
- 阈值 30 → 阈值比例 0.30（30%像素变化触发）
